## 摘要
非结构化剪枝产生稀疏权重张量，但标准实现保持张量形状不变，因此部署的模型并不比剪枝前更小。我们提出一种精确的结构重写方法，称为最小化，它将掩蔽网络转换为更小的密集网络，且其前向函数在浮点数舍入误差范围内保持不变。压缩-释放循环迭代执行剪枝和最小化，并在中间插入一个释放步骤，该步骤将压缩张量内的精确零位置重新激活为小的校准噪声，从而将原本浪费的容量重新变为可训练参数。连续循环利用该容量来发现单次剪枝无法达到的结构冗余。我们还引入了补偿层归一化（CompensatedLayerNorm），这是一种保持功能的层归一化替代方案，它将最小化扩展到跨具有层归一化的残差流的通道缩减。压缩-释放将可部署网络压缩到比未剪枝模型小39倍（在全连接模型网络上）和14.8倍（在现代CNN（ConvNeXt-Tiny）上），且精度相当。此外，我们证明该重写方法可以扩展到Transformer架构。

## Abstract
Unstructured pruning produces sparse weight tensors, but the standard implementation keeps tensor shapes unchanged so the deployed model is no smaller than before pruning. We present an exact structural rewrite, which we call minimization, that converts a masked network into a smaller dense network with the same forward function up to floating-point rounding. The Squeeze-Release cycle iterates pruning and minimization with an intermediate release step that re-enables the exact-zero positions inside the compacted tensors as small calibrated noise, turning otherwise wasted capacity back into trainable parameters. Successive cycles use that capacity to find structural redundancy a single pass cannot reach. We additionally introduce CompensatedLayerNorm, a function-preserving replacement for LayerNorm that extends minimization to channel reduction across LayerNorm-equipped residual streams. Squeeze-Release compresses the deployable network to 39x smaller than the unpruned model on a fully-connected model network and 14.8x smaller on modern CNN (ConvNeXt-Tiny), at comparable accuracy. In addition we prove that the rewrite can be extended to transformer architectures.

## 1. 核心问题与整体含义
- **背景**：非结构化剪枝虽然能产生高稀疏度权重，但标准实现（如 PyTorch 的 `torch.nn.utils.prune`）不改变张量形状，部署模型大小不变，且前向传播仍执行全密集内核，实际加速有限。
- **问题**：剪枝论文通常报告“掩码存活参数数量”（mask-alive count），但该指标不代表可部署模型的实际大小，存在巨大差距（如 ConvNeXt 上可部署参数量是掩码存活数的 11.3 倍）。这误导了其对通用硬件的加速效果。
- **意义**：本文提出 **精确结构最小化（Exact Minimization）**，将剪枝后的掩码网络转换为更小的稠密网络（可部署版本），并引入 **Squeeze-Release 迭代循环** 进一步压缩，同时保持前向函数在浮点误差范围内一致。此外，提出 **CompensatedLayerNorm** 使最小化能处理 LayerNorm 下的通道缩减，扩展至 Transformer 架构。

## 2. 方法论
- **核心思想**：Squeeze-Release 循环，每轮包含四个步骤：
  1. **剪枝**：使用梯度×权重幅度的重要性评分，按神经元/滤波器粒度零化最低分参数，卷积层按滤波器整体评分和移除。
  2. **最小化（Squeeze）**：精确结构重写。识别死输入单元（整行/整个滤波器为零）和死输出单元（整列为零）。通过将死输入单元的恒定输出折叠到下游偏置，然后移除该行/滤波器及其对应列/滤波器，实现网络变小。对 FC 层、Conv2d 滤波器、残差块同样适用。三种模式：A（减少中间 4C 宽度）、B（通过 CompensatedLayerNorm 减少残差流通道）、C（移除整个块）。
  3. **释放（Release）**：将最小化后稠密张量中的精确零位置用小噪声替换（从幸存权重分布采样，乘以 0.01），重新激活为可训练参数，不增加模型大小。文中指出改为零初始化效果类似，说明关键是激活容量而非初始化方式。
  4. **微调**：在相同训练数据上训练固定 epoch。
  - 循环终止条件：验证精度低于阈值或下降超过 10 个百分点；或达到最大循环数。
- **关键技术细节**：
  - **CompensatedLayerNorm**：当移除 LayerNorm 输入中的恒等通道时，存储已移除通道的数量 K、和 S、平方和 Q，利用重新计算的全宽度均值和方差恢复精确保留通道的输出。公式精确，存储开销仅为每层三个标量。
  - **前向偏置折叠**：死输入神经元输出常数，通过 `b_{i+1} ← b_{i+1} + W_{i+1}[:,k] * c_k` 折叠到下游，然后删除对应行/列。
  - 剪枝评分采用 `|∇_θ L · θ|`，对卷积层按滤波器平均评分。
