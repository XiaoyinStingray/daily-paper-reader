## 摘要
面向机器的图像压缩需要一个统一的编解码器来服务于多个下游视觉任务。现有方法要么采用特定任务的端到端设计，导致参数和部署开销增加，要么依赖基于迁移的适应方法，这些方法仍停留在外部附加且基于启发式任务设计。这两类工作的一个共同局限性在于其计算模式基本是静态的，尽管不同图像区域对机器感知具有显著不同的语义重要性和复杂度，但它们仍对所有token应用相似的变换。我们提出MoECodec，这是一个支持在单一模型内完成多个下游任务的token感知图像压缩框架。MoECodec将基于Transformer的压缩模型中的FFN层替换为逐token的混合专家（MoE）模块，实现了基于输入内容和任务目标的动态token级计算。为使MoE在压缩模型中有效，我们引入了一种稳定的路由策略，该策略结合了专家选择路由与空间全变分正则化，以促使空间一致的分配；同时我们提出了一种轻量级专家架构——分组洗牌MLP（GShMLP），以控制参数增长。大量实验表明，MoECodec在传统图像重建和机器任务上均比基线方法有持续改进。

## Abstract
Image compression for machines calls for a unified codec that serves multiple downstream vision tasks. Existing approaches either adopt task-specific end-to-end designs, raising parameter and deployment overhead, or rely on transfer-based adaptations that remain externally attached and heuristic task design. A key limitation shared by both lines of work is their largely static computation pattern, which applies similar transformations across tokens despite the fact that different image regions exhibit markedly different semantic importance and complexity for machine perception. We propose MoECodec, a token-aware image compression framework that supports multiple downstream tasks within a single model. MoECodec replaces the FFN layers in transformer-based compression model token-wise Mixture-of-Experts (MoE), enabling dynamic, token-level computation conditioned on the input content and task objective. To make MoE effective in compression model, we introduce a stable routing strategy that combines expert-choice routing with spatial total variation regularization to encourage spatially coherent assignments, and we propose a lightweight expert architecture, Group Shuffle MLP (GShMLP), to control parameter growth. Extensive experiments show consistent improvement against baselines on both conventional image reconstruction and machine tasks.

## 1. 核心问题与研究动机

- **背景**：机器学习驱动的视觉应用（分类、检测、分割）快速增长，要求图像压缩同时服务于人类视觉和机器感知。传统图像压缩模型针对人类视觉优化（如MSE、MS-SSIM），往往会丢失任务相关的语义信息，降低下游机器任务性能。
- **核心问题**：现有方法存在两个不足：
  - **任务特定端到端设计**：为每个任务训练独立编解码器，参数和部署开销巨大。
  - **基于迁移的适应方法**：在人类导向基模型上加轻量适配模块，但仍然是外部附加、启发式设计，缺乏内在统一优化。
  - **共同局限**：计算模式是静态的，对所有图像token应用相同变换，忽略了不同区域对机器感知的语义重要性和复杂度差异（例如：平坦背景与纹理丰富的物体区域需要不同的计算分配）。
- **目标**：提出一个统一的、支持多种下游任务（重建、分类、检测、分割）的图像压缩框架，实现动态、token级别的条件计算，且单模型即可覆盖多任务。

## 2. 方法论

### 核心思想
- **MoECodec**：将基于Transformer的学习型图像压缩模型（LIC）中的标准前馈网络（FFN）层替换为**逐token的混合专家（MoE）层**，实现内容与任务驱动的动态计算分配。
- **整体架构**：保留原有编码器-熵模型-解码器的自动编码器结构，仅替换所有Transformer块中的FFN为MoE层（如图2a所示）。预训练基模型参数冻结，仅训练路由器和专家网络。

### 关键技术细节

#### 2.1 MoE层设计
- **输入**：中间特征 `X ∈ R^(B×S×d)`（B=batch, S=序列长度=H×W, d=通道维度）。
- **路由网络**：通过可学习投影 `W_r ∈ R^(d×E)` 计算token与E个专家之间的亲和度矩阵：`A = softmax(X W_r, dim=-1) ∈ R^(B×S×E)`。

#### 2.2 路由策略：专家选择（Expert-Choice, EC）路由
- 传统token选择（TC）：每个token独立选择top-k专家，导致负载不均和空间碎片化。
- **EC路由**：每个专家从全部token中主动选择其top-k个token（`k = floor(S·fc/E)`，fc=容量因子），实现：
  - 全局token级竞争，产生更空间一致的路由。
  - 天然负载均衡，无需辅助损失。
- **输出计算**：对每个专家输出加权聚合（公式5）。

#### 2.3 空间总变差正则化（Spatial Total Variance Regularization）
- 将亲和度张量 `A` 重排为 `R^(B×E×H×W)`。
- 对各专家亲和度图施加各向异性TV正则：`L_r = (1/(BE)) Σ_b Σ_e (||∇h Ã_be||_1 + ||∇w Ã_be||_1)`。
- 鼓励相邻token分配到相同专家，抑制路由噪声。仅训练时使用，无推理开销。

#### 2.4 轻量专家架构：分组洗牌MLP（GShMLP）
- **问题**：标准FFN参数为 `8d^2`，E个专家参数量增大E倍。
- **解决方案**：将通道分为G组，每个组内独立进行两个线性投影（块对角矩阵），参数缩减为原来的`1/G`。
- **跨组交互**：在两个投影之间插入无参数**通道洗牌**操作（ShuffleNet v2思想），保持通道混合能力。
- **公式**：`GShMLP(X) = (Shuffle(σ(XW1))) W2`，其中W1、W2为块对角矩阵，W1∈R^(d×4d)，W2∈R^(4d×d)，每组大小`d/G × 4d/G`。

#### 2.5 总训练目标
- `L_total = L_rd + α L_r`，其中`L_rd`为率失真损失，`α=1e-3`。
