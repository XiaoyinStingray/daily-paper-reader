## 摘要
最近的大型视图合成模型（LVSMs）提倡一种编码器-解码器架构，将重建和渲染分离到不同的网络中。我们重新审视了这种设计。通过控制实验，我们展示了仅解码器架构（隐式地将场景表示为KV缓存）在相同渲染复杂度下使用更少的参数，其性能优于编码器-解码器变体。进一步分析表明，在颜色输入重建网络和仅相机渲染网络之间共享权重，能更好地对齐同一视角下的特征，从而促进图像合成。基于这一发现，我们的模型（称为DVSM）进一步融入了基础模型先验和分阶段补丁大小调整，以改进效率-质量权衡。我们的结果在多个基准测试中为新颖视图合成建立了新的最先进水平，在某些情况下，甚至在密集输入视图下优于每场景优化的3DGS。

## Abstract
Recent Large View Synthesis Models (LVSMs) advocate an encoder-decoder architecture that separates reconstruction and rendering into distinct networks. We re-examine this design. Through controlled experiments, we show that a decoder-only architecture, which represents scenes implicitly as a KV-cache, outperforms encoder-decoder variants while using fewer parameters at identical rendering complexity. Further analysis shows that sharing weights between the color-input reconstruction network and the camera-only rendering network better aligns their features at the same viewpoint, facilitating image synthesis. Building on this finding, our model, dubbed DVSM, further incorporates foundation model priors and stage-wise patch sizing for an improved efficiency-quality tradeoff. Our results establish a new state of the art for novel-view synthesis across multiple benchmarks, in some cases even outperforming per-scene-optimized 3DGS under dense input views.

# 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：新颖视图合成（Novel View Synthesis）任务中，近期大型视图合成模型（LVSMs）普遍采用编码器‑解码器架构，将场景重建（Reconstruction）和渲染（Rendering）分离到两个独立的网络中。这种设计存在两点不足：
  1. 编码器学习构建场景时未利用解码器渲染器的知识，两者权重解耦，导致特征对齐差。
  2. 解码器仅变体（Decoder‑only LVSM）为每个新视角独立处理上下文，计算冗余且效率低。
- **研究动机**：重新审视编码器‑解码器设计的必要性，探索更高效的仅解码器架构，使其在保持高质量的同时降低复杂度。

# 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：
  - **隐式场景表示**：将场景表示为跨注意力层的 **KV 缓存（KV‑cache）**，重建阶段计算并缓存，渲染阶段直接查询，避免为每个新视角重复计算。
  - **全权重共享**：强制重建阶段和渲染阶段共用同一套网络权重（包括输入 patch 嵌入层、注意力层、FFN 等），使两个阶段的特征在同一视角下高度对齐。
- **关键技术细节**：
  1. **输入处理**：
     - 重建 token：`LayerNorm(PE_ray(Plücker(P,K)) + PE_rgb(I))`
     - 渲染 token：`LayerNorm(PE_ray(Plücker(P',K')))`
     - PE_ray 和 LayerNorm 权重在阶段间共享。
  2. **架构**：单个解码器 ViT，包含 L 个重复块，每个块由 **帧内注意力（Intra‑view Attention）** 和 **跨视图注意力（Cross‑view Attention）** 组成（均含残差连接和 QK 归一化）。重建阶段缓存所有跨视图注意力的 K、V；渲染阶段仅用目标 token 作为查询访问缓存。
  3. **输出**：最后层 token 经线性投影和像素混洗（PixelShuffle）得到最终图像 `¯I`。
  4. **基础模型先验注入**：在重建分支中额外叠加 DINOv3 特征（`Prior(I)`），通过线性投影对齐维度。
  5. **分阶段 patch size**：重建和渲染使用不同 patch 大小（如 `p1` 和 `p2`），通过缩放图像统一输入分辨率，共享同一 patch 嵌入层。

1. **全权重共享的仅解码器架构最优**：任何形式的权重解耦（包括输入投影层）均导致 PSNR 下降，完全解耦（编码器‑解码器）降幅最大（‑1.41 dB），尽管参数量更大。
2. **跨视图注意力对质量至关重要**：去除后重建速度提升 60%，但 PSNR 损失达 1.6 dB。
3. **基础模型先验显著提升质量**：注入 DINOv3 特征带来 +0.5~+1.0 dB 的 PSNR 增益，且不增加渲染开销。
4. **分阶段 patch size 有效平衡效率‑质量**：重建用大 patch、渲染用小 patch 可在相近渲染速度下获得显著质量提升。
5. **达到 SOTA**：在 Re10K 两视图、DL3DV 多视图、零样本泛化、ScanNet++ 外推等设定下均超越以往所有方法，部分场景（64 视图）质量已匹配或超过每场景优化的 3DGS（使用 200‑400 视图）。

# 8. 不足与局限

- **渲染速度仍不足**：即使采用 ps16，FPS 仅 15‑40，远未达到实时（而 3DGS 可达 >50 FPS），仍落后于几何方法。
