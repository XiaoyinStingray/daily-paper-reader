## 摘要
基于大型语言模型（LLM）的无损图像压缩方法通常通过预训练模型的原生文本接口表示像素数据，将像素值转换为LLM通过其词表头处理的token序列。这种设计表明，预训练语言模型可以为图像编码提供概率估计，但也将压缩与分词器行为、词表特定的数值token以及模型家族特定的适应耦合在一起。本文提出LUMI（基于LLM的统一模型无关无损图像压缩），一种与分词器无关的框架，用于在冻结LLM骨干的情况下进行RGB无损图像压缩。LUMI使用像素嵌入模块替代“像素即文本”的token化方式，将原始强度与通道信息映射到LLM的连续嵌入空间。它进一步引入块内位置编码以保留展开后的二维空间结构，并使用256路预测头产生原生像素字母表的概率。仅训练像素嵌入、位置编码、软前缀参数和预测头，而LLM骨干保持固定。在自然、医学和遥感图像基准上，使用LLaMA、Qwen和Gemma骨干进行的实验表明，LUMI提供了跨分词器家族的统一接口，实现了有竞争力的压缩率，并提升了基于tokenizer的LLM压缩基准的跨域鲁棒性。这些结果将基于LLM的无损图像压缩表述为冻结基础模型在像素空间中的适应，而非特定于分词器的语言符号建模。

## Abstract
Large language model (LLM)-based lossless image compression methods typically represent pixel data through the native text interface of a pretrained model, converting pixel values into token sequences that the LLM processes through its vocabulary head. This design shows that pretrained language models can provide probability estimates for image coding, but it also couples compression to tokenizer behavior, vocabulary-specific numeric tokens, and model-family-specific adaptation. In this paper, we present LUMI (LLM-based Unified Model-agnostic lossless Image compression), a tokenizer-agnostic framework for lossless RGB image compression with frozen LLM backbones. LUMI replaces pixel-as-text tokenization with a pixel embedding module that maps raw intensity and channel information into the continuous embedding space of the LLM. It further introduces intra-patch position encoding to retain two-dimensional spatial structure after flattening, and uses a 256-way prediction head to produce probabilities over the native pixel alphabet. Only the pixel embedding, position encoding, soft-prefix parameters, and prediction head are trained, while the LLM backbone remains fixed. Experiments on natural, medical, and remote-sensing image benchmarks with LLaMA, Qwen, and Gemma backbones show that LUMI provides a unified interface across tokenizer families, achieves competitive compression rates, and improves cross-domain robustness over tokenizer-based LLM compression baselines. These results formulate LLM-based lossless image compression as pixel-space adaptation of frozen foundation models rather than tokenizer-specific language-symbol modeling.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
LUMI 提出一种与 tokenizer 无关的接口：在输入侧，用像素嵌入模块（PixEmb）将原始像素强度、通道标识映射到 LLM 的连续嵌入空间；在输出侧，用 256 路预测头（Pixel Head）直接输出像素字母表上的概率分布。同时引入块内位置编码（INP）恢复展平后的二维空间结构，并添加可训练的软前缀（Soft Prefix）和固定任务提示（Task Prompt）。整个过程中 LLM 骨干保持冻结，仅训练外部轻量模块。

### 关键技术细节
1. **像素嵌入模块（PixEmb）**：
   - 对于第 t 个子像素，构造 7 维描述子 a_t，包含：
     - 归一化强度 u_t = x_t/255 及其中心化、平方项；
     - 三角函数特征 sin(2πu_t), cos(2πu_t)（类似傅里叶特征）；
     - 通道独热编码 e_{κ_t}（κ_t ∈ {0,1,2} 表示 R/G/B）。
   - 通过两层 GELU MLP（7→2d→d）映射到 LLM 嵌入维度 d，得到 e^{pix}_t。

2. **块内位置编码（INP）**：
   - 为每个行索引 r 和列索引 s 分别学习 d 维可训练嵌入表 R∈R^{H_p×d} 和 C∈R^{W_p×d}。
   - 子像素位置向量为 s^{pos}_t = β(R_{r_t} + C_{s_t})，β 为可学习标量门控。
   - 最终像素表示 z_t = e^{pix}_t + s^{pos}_t。

3. **冻结 LLM 熵模型**：
   - 输入序列 E_aug = [τ; E_text(q); Z_{<T}]，其中 τ 为可训练软前缀（长度 P=16），E_text(q) 为固定任务提示的文本嵌入，Z_{<T} 为前 T-1 个像素表示。
   - 冻结 decoder-only LLM 处理得到隐状态 H，取对应位置 h_t 输入预测头。
   - 256 路预测头（两层 GELU MLP d→256）输出概率分布 π_t = softmax(Head_ω(h_t))，供算术编码使用。

4. **训练目标**：
   - 仅优化外部参数 A = {ϕ (PixEmb), ψ (INP 行/列表与门控), ω (预测头), τ (软前缀)}，冻结 LLM 骨干 θ。
   - 使用教师强制和交叉熵损失（256 类）最小化负对数似然。

5. **算术编码与解码**：
   - 编码器和解码器共享相同的确定性概率模型（由相同冻结 LLM 和训练参数产生），逐符号编码/解码，保证无损。

## 6. 论文的主要结论与发现

1. **Tokenizer 无关性**：LUMI 在不同 LLM 家族（LLaMA, Qwen, Gemma）上表现一致，避免了 tokenizer 导致的序列长度不一致问题。
2. **竞争性压缩性能**：在 Kodak, BRACS, BED4RS 上，LUMI 的 BPP 优于 JPEG-XL 和 DLPR（平均 8.91 vs 9.29 vs 9.44），且超越基于 tokenizer 的 P²-LLM LoRA 等基线。
3. **强跨域泛化**：在 leave-one-out 域外测试中，LUMI 仍保持良好性能（如 Kodak 8.68 BPP，BRACS 10.15 BPP），优于 P²-LLM LoRA。
4. **轻量适应有效**：仅训练极少量外部参数（PixEmb, INP, 预测头, 软前缀），即可利用冻结 LLM 的序列建模能力。
5. **可扩展性**：增大冻结骨干模型容量或增加训练数据，压缩性能持续提升，表明 LUMI 是一个可扩展的框架。
6. **输入空间重编程**：将无前缀图像压缩视为对 LLM 的输入重编程，任务提示和软前缀共同贡献。

## 7. 优点：方法或实验设计上的亮点

- **创新性**：首次提出 tokenizer 无关的 LLM 无损图像压缩接口，彻底剥离对文本 tokenizer 的依赖，方法简洁且通用。
- **模块化设计**：PixEmb、INP、预测头均为轻量可插拔模块，可移植到任何 decoder-only LLM，无需修改模型权重。
- **冻结骨干优势**：避免大规模微调（LoRA/全微调），降低训练成本；保留 LLM 作为共享计算引擎的可能性。
- **实验全面性**：覆盖多领域（自然/医学/遥感）、多 LLM 家族、多骨干规模，并设计域外泛化和跨域迁移实验，证明了鲁棒性。

## 8. 不足与局限

- **缺乏与现代深度学习 lossless codec 的全面对比**：仅对比了 JPEG-XL 和 DLPR，未与 iVPF、iFlow、Bit-Slicing 等更优学习型 codec 比较。
- **独立补丁处理**：当前框架将图像切分为非重叠 16×16 补丁单独处理，无法利用补丁间跨块上下文，限制了压缩率。
- **推理延迟高**：LLM 自回归推理步骤多（每个子像素一次前向），解码速度远低于传统 codec（论文自身提及）。
- **骨干预测能力上限**：由于 LLM 骨干冻结，压缩性能受限于 LLM 本身对像素序列的预测能力；若 LLM 在图像领域先验不足，性能可能受限。
- **训练设置限制**：仅使用 12 个 epochs 和少量训练数据（如 Kodak 仅 12 张训练图像），可能未完全发挥模型潜力；但论文展示了数据规模增长提升性能。
- **未充分探讨不同 backbone 间的差异**：虽然三个骨干表现相近，但未分析为何 Gemma 略差于 LLaMA/Qwen，以及 LLM 预训练数据对压缩性能的影响。
- **缺乏对超参数（如 patch size, soft prefix 长度）的敏感性分析**：固定为 16×16 和 P=16，未验证其最优性。
