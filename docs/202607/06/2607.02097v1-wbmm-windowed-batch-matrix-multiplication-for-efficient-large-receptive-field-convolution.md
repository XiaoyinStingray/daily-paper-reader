## 摘要
大卷积核深度可分离卷积虽性能强劲，但随着核尺寸增大，基于收集的计算导致不规则内存访问，性能显著下降；而大卷积核加速（LKA）在小型特征图上有效，在大型特征图上却适得其反，甚至慢于未加速的实现。我们提出窗口化批量矩阵乘法（WBMM），它将输入划分为连续窗口，并索引一个紧凑的相对位置偏置表来构建权重矩阵，通过批量矩阵乘法实现规则内存访问。这带来一个独特特性：WBMM的吞吐量随窗口增大而提升，与深度可分离卷积性能随核增大而下降相反。运算符级基准测试表明，14×14窗口的WBMM在速度上优于5×5深度可分离卷积基线，同时每层感受野扩大7.8倍。结合块间跨窗口通信与分级窗口重参数化，WBMM在ImageNet-1K、COCO和ADE20K上取得相当或更高的精度，训练速度提升1.31–1.88倍，并在GPU、CPU和边缘设备上展现一致优势，无需专用加速核。我们的代码见http://github.com/wansong-s/WBMM

## Abstract
Large kernel depthwise convolutions achieve strong performance but suffer from significant degradation as kernel size grows due to irregular memory access from gather-based computation; while Large Kernel Acceleration (LKA) helps on small feature maps, it becomes counterproductive on large feature maps, even slower than non-accelerated implementations. We propose Windowed Batch Matrix Multiplication (WBMM), which partitions input into contiguous windows and indexes a compact relative position bias table to construct weight matrices, enabling regular memory access via batched matrix multiplication. This yields a unique property: WBMM's throughput improves with larger windows, opposite to depthwise convolutions that degrade with larger kernels. Operator-level benchmarks show WBMM with 14x14 windows outperforms 5x5 depthwise convolution baselines in speed while providing a 7.8x larger per-layer receptive field. Combined with inter-block cross-window communication and hierarchical window reparameterization, WBMM achieves comparable or higher accuracy on ImageNet-1K, COCO, and ADE20K with 1.31-1.88x training speedup, and demonstrates consistent advantages across GPU, CPU, and edge devices without requiring specialized acceleration kernels. Our code is available at http://github.com/wansong-s/WBMM

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：大卷积核深度可分离卷积（如7×7、13×13以上）虽然在视觉任务中表现优异，但其基于“收集（gather）”的计算模式导致不规则内存访问，随着核尺寸增大，性能严重退化（从5×5到13×13吞吐量下降71–78%）。
- **现有加速的局限**：Large Kernel Acceleration (LKA) CUDA kernel 在小特征图上有效，但在大特征图上（如224×224）反而比未加速版本更慢（7×7核慢80%，13×13核慢89%），因为其固定分块策略不匹配大分辨率。
- **整体含义**：需要一种新的计算范式，在不牺牲感受野的前提下，实现规整、可伸缩的内存访问，从而让大感受野卷积在高分辨率场景下变得高效。

## 2. 方法论：核心思想、关键技术细节
### 核心思想
- **改变计算范式**：传统方法遍历输入数据，收集邻域；WBMM则**遍历参数表**，通过索引构建权重矩阵，再对连续窗口执行批量矩阵乘法，实现规整内存访问。
- **关键洞察**：WBMM的吞吐量随窗口增大而提升（因为算术强度O(w²)随窗口增大而增加），而深度卷积正好相反。

### 关键技术细节
1. **窗口划分与权重构建**：
   - 将输入划分为非重叠的连续窗口（如7×7）。
   - 使用紧凑的相对位置偏置表 `R ∈ R^{C×(2w_h-1)(2w_w-1)}`，通过预计算的位置索引矩阵 `I` 索引得到权重矩阵 `M ∈ R^{C×d×d}`（d=w_h·w_w）。
   - 输入窗口与M执行批量矩阵乘法，得到输出。
2. **批无关权重构建**：权重M由通道参数决定，与批次和样本无关，可被整个批次共享，在推理时缓存。
3. **处理任意输入尺寸**：通过零填充使H、W可被窗口尺寸整除，计算后移除填充。
4. **跨窗口通信**：使用轻量3×3深度卷积作为独立块，与WBMM块交替部署（inter-block设计），避免将3×3与WBMM放入同一块，以保持kernel优化的简洁性。
5. **分级窗口重参数化**：对密集预测任务，训练时使用全局大窗口（14×14）和局部小子窗口（7×7）两条独立分支，推理时融合为单一矩阵，不增加推理开销。
6. **小特征图多核融合**：在最小特征图（S4阶段）添加并行深度卷积路径（5×5、3×3），推理时融合到WBMM权重中。

- **WBMM随窗口增大而加速**（14×14优于7×7），与深度卷积退化相反。

## 7. 优点（方法或实验亮点）
1. **创新计算范式**：从“遍历输入数据”转为“遍历参数表”，从根本上解决不规则内存访问。
2. **独特逆向缩放属性**：WBMM吞吐量随窗口增大而提升，与直觉和传统卷积相反。
3. **批无关权重构建**：权重矩阵M在批次内共享，推理时可缓存，大幅提升batch scalability。
4. **实用化设计**：inter-block 3×3通信、分级重参数化、零填充策略、多核融合等组件轻量有效。
5. **跨架构通用性**：不仅在UniRepLKNet上验证，也在ConvNeXt-T上成功替换，证明方法不依赖特定架构。
6. **实验全面且公平**：严格对照UniRepLKNet代码库，控制变量；涵盖5个模型尺寸、3个任务、多种硬件，消融实验细致。

## 8. 不足与局限
- **超大窗口的内存成本**：当窗口>28×28时，权值矩阵M ∈ R^{C×d×d} 的显存占用迅速增长（O(w⁴)），可能变得不实用。论文指出可通过Flash-Attention风格分块核来缓解，但尚未实现。
- **输入无关性**：WBMM的权重固定，不依赖于输入内容，而某些需要内容自适应路由的任务（如VQA、卫星图像）可能从输入相关权重中获益。
- **窗口尺寸需人工选择**：实验表明纯14×14并不总优于7×7，需依赖分级重参数化或手动调优，自动化窗口搜索是未来方向。
- **扩展至1D/3D未验证**：论文仅聚焦2D视觉，提出抽象可扩展但未在时序或视频数据上实验。
- **实验缺乏大规模预训练**：Base模型在ImageNet-1K上精度饱和（83.9%），未使用ImageNet-22K预训练，与UniRepLKNet-B的对比仅在1K上。
- **潜在偏差风险**：所有实验基于UniRepLKNet代码库，虽然进行了跨架构验证，但可能仍受益于原库的总体设计；边缘设备测试仅限Orin Nano，未覆盖更多低功耗硬件。
