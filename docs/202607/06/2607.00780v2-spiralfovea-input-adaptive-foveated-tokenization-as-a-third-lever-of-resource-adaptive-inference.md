## 摘要
大多数面向基础模型的自适应推理技术改变了模型的行为——提前退出、MoE路由、KV缓存压缩、动态注意力稀疏。然而，输入到骨干网络的数据仍然是固定网格的标记化，与图像内容无关。我们认为这是一个被忽视的杠杆。我们提出了SpiralFovea，一种无参数、输入自适应的标记器，其中标记的身份、位置、尺度和数量都是局部视觉熵的函数，并且在查询任何骨干参数之前完成选择。围绕内容驱动的热点锚点，多尺度螺旋环产生≤78个补丁，在输入阶段替换标准的196补丁ViT网格。在四个经典的细粒度基准上，SpiralFovea实现了+1.7-2.1个百分点的准确率提升，同时输入标记减少60%，每个Transformer层的自注意力FLOPs减少84%，吞吐量相比匹配的静态标记化基线提高18-29%。在CUB-200-2011 Genus上跨越四个骨干网络的受控消融实验揭示了一个清晰的诊断：增益幅度与骨干网络的全图位置先验强度呈反比，从而将自监督基础模型隔离为输入自适应标记化最有价值的场景。

## Abstract
Most adaptive-inference techniques for foundation models change what the model does - early exit, MoE routing, KV-cache compression, dynamic attention sparsity. The input that hits the backbone, however, remains a fixed-grid tokenisation indifferent to image content. We argue that this is a missed lever. We present SpiralFovea, a parameter-free, input-adaptive tokeniser in which token identity, location, scale, and count are all functions of local visual entropy and selection completes before any backbone parameter is queried. Around content-driven hotspot anchors, multi-scale spiral rings produce <= 78 patches that replace the standard 196-patch ViT grid at the input stage. Across four canonical fine-grained benchmarks, SpiralFovea yields +1.7-2.1 pp accuracy with a 60% reduction in input tokens, an 84% reduction in self-attention FLOPs at every transformer layer, and 18-29% throughput gains over the matched static tokenisation baseline. A controlled ablation on CUB-200-2011 Genus across four backbones reveals a clean diagnostic: the gain magnitude tracks inversely with the strength of the backbone's whole-image positional prior, isolating self-supervised foundation models as the regime where input-adaptive tokenisation is most valuable.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：当前资源自适应推理方法（如提前退出、MoE路由、KV缓存压缩、动态注意力稀疏）主要改变模型的计算路径，但**输入到骨干网络的标记化过程仍然是固定网格且与图像内容无关**。这违背了最优比特分配原则：应根据局部信息密度分配计算资源，而均匀标记化对所有区域一视同仁，导致计算浪费。
- **整体含义**：本文提出“输入自适应标记化”作为**资源自适应的第三杠杆**，与架构/深度杠杆（杠杆1）、注意力/缓存杠杆（杠杆2）互补。该杠杆在骨干网络执行任何计算之前，根据图像内容动态决定哪些token存在、在哪里、多大尺度以及多少个，从而实现**严格帕累托改进**：更高准确率 + 更少计算量。
- **背景**：人类视觉系统通过中央凹集中高分辨率采样于高信息区域，外围处理用于引导注视。SpiralFovea 模拟这一生物机制，使用局部视觉熵驱动标记化，无需可学习参数。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- **内容驱动标记化**：token的身份、位置、尺度、数量均由图像局部视觉熵函数决定，且在查询任何骨干参数前完成选择。与事后剪枝（如DynamicViT）不同，SpiralFovea 不会为无用token创建并处理任何中间表示。

### 关键技术细节
1. **熵图计算**：
   - 将输入图像转为灰度（BT.601），下采样至 D×D（D=112），量化至 B=16 个bin。
   - 使用滑动窗口（半径 r，ω=2r+1=15）计算局部经验概率 \(\hat{p}_b(x,y)\)，再求局部香农熵：
     \[
     E(x,y) = -\sum_{b=0}^{B-1} \hat{p}_b(x,y) \log_2(\hat{p}_b(x,y)+\varepsilon)
     \]
   - 双线性插值上采样至原始分辨率 H×W。计算开销 < 单个Transformer块的2%。

2. **热点锚点检测**：
   - 将图像高度分为 S=8 条带，对每个活跃条带（s∈{0,2,4,6}）取熵最大的位置：
     \[
     c_s = \arg\max_{(x,y)\in\Omega_s} E(x,y)
     \]
   - 条带分解防止所有锚点坍缩到全局最大值。
   - 锚点归一化至[-1,1]²后经重复去除（τ_dedup=0.15），保证锚点集为τ_dedup/2-填充（证明见定理2.1）。

3. **多尺度螺旋环抽取**：
   - 在每个保留锚点周围布置 K=4 个同心环，半径递增、补丁尺寸递增。
   - 环k=0为单个中心补丁；环k>0上放置 n_k 个补丁，角度均匀分布，半径和间距通过预先设计的调度参数控制（224像素画布上：[(24,0),(28,18),(36,22),(48,26)]）。
   - 每个锚点理论上最多产生 29 个补丁（1+5+10+13），4个锚点上限116个，但大量外环补丁超出图像边界被丢弃，实际 ≤78 个（约减少60%）。

4. **Token获取与嵌入**：
   - 补丁通过双线性网格采样（grid_sample）按 Pr×Pr （ViT为14×14）采样，丢弃越界分数超过 τ_dedup=0.6 的补丁。
   - 使用骨干网络冻结的 patch_embed.proj 投影。
   - **位置编码**：采用从极坐标（中心锚点、环索引、角度）学到的2层MLP（极性位置编码），比正弦光栅编码更适合螺旋布局的几何邻近性。
   - CLS token 拼接后送入完整冻结Transformer堆栈，最终 CLS token 经过线性头分类。

5. **CNN融合（Mamba）**：对于CNN骨干（如ResNet-18），每个32×32补丁通过共享ResNet-18主干编码，再用两个Mamba块融合可变长token序列（O(N)复杂度，适应图像间token数变化）。

## 6. 论文的主要结论与发现

1. **严格帕累托改进**：在所有4个基准和4个骨干上，SpiralFovea 同时提升了准确率（+1.7~2.1pp）并减少了计算量（60% input tokens, 84% self-attention FLOPs，18-29%吞吐量提升）。

## 7. 优点

1. **创新的第三杠杆**：将输入标记集本身作为可调节资源杠杆，概念清晰且正交于现有方法。
2. **无参数、高效**：熵计算和锚点检测无学习参数，节省计算且避免过拟合风险。
3. **严格前期筛选**：在第一个骨干参数计算前完成选择，与事后剪枝形成本质区别。
4. **生物启发设计**：多尺度螺旋环模拟中央凹分布，与人类视觉系统高度契合。
5. **全面实验与诊断**：包含多个骨干、基准、消融、效率、边界分析，实验结果一致且可解释。
6. **兼容性**：可直接插入冻结的开箱即用骨干（ViT/DINO），仅需训练小量额外参数。

## 8. 不足与局限

1. **锚点多样性有限**：条带分解仅强制水平多样性；使用学习型2D显著性估计器可能更通用。
2. **可变序列长度**：每个batch需要padding以适应不同图像token数，影响批处理效率；桶式批处理（bucketed batching）未探索。
3. **测试范围有限**：仅评估细粒度识别（空间集中信号明显）。论文指出在ImageNet整图分类上预期增益中性至轻微正面（需在DINO自监督骨干下验证），但未提供实验证实。
4. **边界案例存在负增益**：监督ViT-S/16在CUB-Genus上-0.28pp，论文归因于饱和和强先验，但表明方法并非通用增益。
5. **计算开销无法完全消除**：虽然熵图计算小于一个Transformer块，对于实时场景可能仍有瓶颈；且grid sample采样操作本身有成本。
6. **仅适用于静态图像**：未扩展至视频或非图像模态（文本、音频、代码），尽管论文提到这是开放问题。
7. **可叠加性未实证**：声称与杠杆1、2可叠加产生乘法节省，但未进行叠加实验验证。
8. **对比方法缺失**：未与同样是输入自适应的像素级注视采样（如Recasens2018）直接对比，也未与剪枝方法（DynamicViT等）在相同设置下比较性能（强调方法本质不同，但缺乏兼容性比较）。
