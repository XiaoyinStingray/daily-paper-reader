## 摘要
离散视觉分词器将图像转换为有序的编码序列，为场景的结构化描述提供了一种自然的表示。然而，现有的自适应分词器要么需要事后搜索，要么从一组离散的预训练率中进行选择，而不是学习一个与模型和场景耦合的每图像连续序列长度，并且它们通常针对像素重建进行训练，强调纹理而非结构。我们提出STROP，一种离散视觉分词器架构，它形成结构化场景表示，同时学习图像视觉程序应有的长度。通过使用由局部率-失真探针监督的四阶段课程，并以冻结的DINOv3特征为基准，STROP优化了一个专用的长度头，该头在单次前向传递中估计活动的 prefix 长度。通过绕过像素级重建梯度，码本完全由更高级潜在表示的质量塑造。程序长度随场景复杂度增长，并且在下游密集预测迁移和直接检查学习到的编码词汇表中，都出现了组合性结构的迹象。

## Abstract
Discrete visual tokenizers translate images into ordered sequences of codes, providing a natural representation for structural description of scenes. Yet existing adaptive tokenizers either require post-hoc search or select among a discrete set of pre-trained rates, rather than learning a continuous per-image sequence length coupled to the model and scene, and they typically train against pixel reconstruction, emphasizing texture rather than structure. We propose STROP, a discrete visual tokenizer architecture that forms structural scene representations and simultaneously learns how long an image's visual program should be. Using a four-phase curriculum supervised by local rate--distortion probes against frozen DINOv3 features, STROP optimizes a dedicated length head that estimates the active prefix length in a single forward pass. By bypassing pixel-level reconstruction gradients, the codebook is shaped entirely by the quality of higher-level latent representations. Program length grows with scene complexity, and signs of compositional structure emerge both in downstream dense-prediction transfer and in direct inspection of the learned code vocabulary.

### 论文核心问题与整体含义
本研究的核心动机在于指出当前离散视觉分词器（DVTs）在构建场景结构化表示时的两大不足：
- **固定的序列长度**：多数分词器（如VQ-VAE、VQGAN、TiTok）为每张图像分配固定数量的编码（即“视觉程序”长度固定），这不符合“奥卡姆剃刀”等原则——简单场景与复杂场景应使用不同长度的描述。
- **像素重建目标的偏向性**：现有训练目标多以像素级保真度为主，这易导致模型浪费编码预算在纹理细节上，而非捕获对场景理解更关键的语义或几何结构。
整体而言，论文旨在实现一种**结构优于像素**的视觉程序学习——让模型能根据图像内容的复杂度，自适应地生成可变长度的、富含结构信息的离散编码序列。

### 方法论
论文提出了 **STROP（Structure over Pixels）** 架构，其核心创新在于**长度自主学习的离散视觉分词器**，主要包含以下关键技术细节：

- **架构组成**：
  - **生成器-解释器对**：一个编码器（生成器）将冻结的DINOv3 ViT块特征映射为有序的编码序列（视觉程序），并通过向量量化（VQ）瓶颈转换为离散编码；另一个编码器（解释器）则根据这些离散编码重建回块级特征场。
  - **长度预测头**：一个轻量级MLP，以编码器特征的全局池化作为输入，直接预测当前图像所需的程序长度`L`。

- **创新的训练课程**：
  为了端到端地训练长度预测头，避免事后搜索或固定速率选择，论文设计了一个四阶段训练课程：
  1. **随机截断（Random truncation）**：训练时随机抽取程序前缀长度，迫使模型将关键信息前置到较早的编码中。长度偏向均匀分布。
  2. **Oracle目标估计**：继续随机截断，但开始为每个样本计算一个“理想长度”（Oracle），通过分析重建误差及其局部梯度（判断缩短/延长前缀的收益）来确定。
  3. **监督式长度头训练**：激活长度预测头，以第二阶段积累的理想长度为监督目标进行训练，但此时仍采用随机截断，保证头训练的稳定性。
  4. **交接（Handoff）**：以逐渐增大的概率从随机截断切换为模型自预测的长度截断，实现平滑过渡。推理时，程序直接根据预测长度进行硬截断。

- **核心监督信号**：训练主损失`L_lat`是重建的块特征与冻结DINOv3教师特征之间的**余弦相似度 + 均方误差（MSE）** 的组合。完全**不依赖像素级重建损失**，从而引导码本学习更高级的语义结构。像素解码器仅作为可视化检查工具，其输入来自解释器输出的停止梯度（stop-gradient）特征。

1.  **灵活的长程序成功学得**：STROP能成功预测出与场景复杂度正相关的程序长度（如在CLEVR中与物体数量相关系数达0.74）。

3.  **程序具备组合结构**：通过令牌消除归因实验发现，单个程序编码像“语义手柄”一样，能局域地、可解释地影响特定的语义区域，表明编码间存在空间对应关系。
4.  **表征等级差异**：通过解释器映射回的`Latent`表征在下游任务中远优于直接使用`Program`表征，表明程序更多是一种可执行的中间表示，而非直接适配线性探针的非结构化向量。

### 优点
- **新颖的训练范式**：提出的四阶段课程学习**无需后处理搜索**、无需预定义速率集，即可在单次前向传播中端到端地学习预测每图像的连续长度，这是对现有方法的显著改进。
- **目标函数的创新**：彻底**绕过像素重建损失**，完全以DINOv3语义特征为监督信号，为学习结构化视觉表示探索了一条更优的路径。
- **全面的可解释性分析**：通过反事实消除、令牌协同、LLM解码等多种方式，深入探查了学到的离散程序的内在结构、组合性和可解释性，分析维度丰富。

### 不足与局限
- **原始程序的可读性不足**：直接的`Program`表征在线性探针任务中性能不佳，说明离散编码序列作为直接可用特征仍存在困难。论文将此归咎于探针过于浅层，但无法完全排除量化过程的信息损失。
- **任务间性能不一致**：在Cityscapes分割任务上，STROP并未优于FlexTok。可能预示该方法在处理精细、高密度街景时存在局限，或教师模型、训练数据的分布影响了泛化性。
- **属性绑定的缺失**：LLM从CLEVR编码解码场景的实验中，模型能较好恢复物体位置和数量，但**属性（如颜色、材质）解码准确率未超过统计基线**，表明当前码本未能将物体属性稳固地绑定到特定编码上。
- **训练开销**：课程学习比固定长度训练慢约10%，且需要仔细调整多阶段超参数。
