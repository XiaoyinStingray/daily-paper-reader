## 摘要
媒体压缩标准在率失真复杂度权衡方面已达到瓶颈，限制了在机器人、可穿戴设备和遥感等应用中将昂贵的AI感知卸载到云端的能力。基于DNN的编解码器提高了压缩效率，但代价是：它们难以轻松适应可用比特率的较大变化，并且实时编码需要昂贵且功耗高的GPU，这使得在低成本或资源受限的平台上无法使用。为解决这些限制，我们提出了一种新颖的自编码框架（FRAPPE），它使用全输入通过投影追踪编码器预测残差输出。FRAPPE的编码目标自然地将潜在通道按重要性排序，从而实现零开销的变速率编码。与基于RNN的学到的编解码器（其编码器消耗前一重建的残差）或RVQ风格的编解码器（其码本必须顺序应用）不同，FRAPPE的分析路径是一个由独立输入投影组成的易于并行化的有向无环图（DAG）。利用FRAPPE，我们构建了一个变速率RGB图像编解码器（FRAPPE-Image），并评估了其与标准图像编解码器相比的率失真复杂度权衡。在高压缩比（约0.1 bpp）下，FRAPPE-Image提供了比AVIF更高的感知质量，且编码速度快47倍，使其能够仅用CPU实现实时1080p、30fps编码。我们的代码和预训练模型已开源：https://github.com/UT-SysML/FRAPPE。

## Abstract
Media compression standards have reached a plateau in terms of the rate-distortion-complexity trade-off, limiting the ability to offload expensive AI perception to the cloud in applications like robotics, wearables, and remote sensing. DNN-based codecs improve compression efficiency, but at a cost: they cannot easily adapt to large changes in available bitrate, and real-time encoding requires expensive, power-hungry GPUs that prohibit use on low-cost or resource-constrained platforms. To address these limitations, we propose a novel autoencoding framework (FRAPPE) that uses the Full input to predict the Residual output via a Projection Pursuit Encoder. FRAPPE's encoding objective naturally sorts latent channels by importance, allowing zero-overhead variable-rate coding. Unlike RNN-based learned codecs, whose encoder consumes the previous reconstruction's residual, or RVQ-style codecs, whose codebooks must be applied sequentially, FRAPPE's analysis path is an embarrassingly parallel DAG of independent input projections. Using FRAPPE, we build a variable-rate RGB image codec (FRAPPE-Image), and evaluate its rate-distortion-complexity trade-off against standard image codecs. At high compression ratios (approx. 0.1 bpp) FRAPPE-Image provides higher perceptual quality than AVIF with 47 times faster encoding, making it capable of real-time 1080p, 30fps CPU-only encoding. Our code and pre-trained models are available: https://github.com/UT-SysML/FRAPPE .

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：传统图像/视频压缩标准（VVC、AV1等）在率失真-复杂度权衡上已进入平台期，且编码成本高、功耗大；基于深度神经网络（DNN）的学习型编解码器虽然提升了压缩效率，但存在两大瓶颈：
  1. 难以灵活适应大范围比特率变化（需重新训练或复杂调节）；
  2. 实时编码依赖昂贵、高功耗的GPU，不适用于资源受限的传感器、机器人、可穿戴设备等场景。
- **目标**：提出一种同时满足以下条件的框架：（1）零开销变速率/渐进式编码；（2）在极低比特率下达到有竞争力的率失真性能；（3）可在无GPU加速的CPU上实现实时编码。

## 2. 方法论：核心思想与关键技术细节
### 2.1 核心思想
- **全输入预测残差输出**（Full Input, Residual Output）：编码器始终接收原始全分辨率输入 \(x\)，但训练目标是拟合前 \(m-1\) 个通道重建后的残差 \(r_m = x - \hat{x}_{m-1}\)。这不同于传统循环网络（需消耗前一重建的残差）或RVQ（需顺序量化），使得各通道投影可并行计算。
- **投影追踪编码器**（Projection-Pursuit Encoder）：逐通道训练，每个新通道的线性投影方向 \(\omega_m\) 和随后的非线性函数（由合成变换联合实现）被优化以最大限度地减少输出空间残差。训练时自然按重要性排序潜在通道，编码时只需保留前 \(n\) 个通道即可获得不同码率点。

### 2.2 关键技术细节
- **编码器分析变换** \(G_A\)：由 \(S=5\) 个尺度组组成，每个尺度组包含 \(n_s\) 个通道，每个通道是一个非重叠图像块上的学习线性投影（即有效步长相等的卷积）。所有尺度组的投影在推理时可合并为一个步长卷积层，从而形成易并行化的有向无环图（DAG）。计算量仅约为每像素68次乘加（ops/pixel）。
- **压缩非线性** \(\Phi\)：带学习尺度参数的softsign函数，将输出限制在 \([-127, 127]\) 的有符号8位范围内。
- **量化与熵编码**：采用训练时加噪声、测试时硬取整的量化；熵编码使用JPEG-LS（对companded潜在值进行无失真编码）。
- **合成变换** \(G_S\)：一个深度ConvNeXt风格网络（12个残差块，宽度768），固定解码器分辨率 \(p_d=8\)，将所有尺度组的量化潜在值重新映射并拼接后输入，输出重建图像。
- **训练流程**：分两阶段。
  1. **单通道残差阶段**：按通道顺序训练每个新通道的投影方向和对应的合成变换（只更新新加入的部分），损失函数为 \(L_m = \log_{10} \|r_m - \hat{r}_m\|_2^2 + \lambda_m \cdot \mathbb{E}[r^2_m]^\rho \log_2 \text{Std}(\Phi(\cdot))\)，其中 \(\rho=0.3\) 防止后期残差能量过小时率项主导。
  2. **合并解码器阶段**：冻结所有编码器权重，仅对 \(G_S\) 进行再训练（\(\lambda=0\)），为每个已激活通道数 \(n\) 单独保存一个解码器快照。
- **变速率实现**：编码时只需选择保留前 \(n\) 个通道（无需重新运行编码器），配合对应 \(n\) 的解码器快照即可，实现零开销变速率和渐进式传输。

### 2.3 公式与算法流程（文字说明）
- 整体编解码：\(\hat{x} = G_S \circ \text{Adapt}_{p_d} \circ Q \circ \Phi \circ G_A(x)\)。
- 编码器：\(z^{(s)} = \Phi\left(W^{(s)} *_{p_s} x + b^{(s)}\right)\)，所有尺度并行，之后分别量化和熵编码。
- 训练时逐通道添加，最终得到21个通道按重要性自然排序。

## 6. 主要结论与发现
- **极低比特率下性能领先**：在约0.1 bpp（压缩比240:1）时，FRAPPE-Image的感知质量（DISTS）优于AVIF，且编码速度快47倍。
- **实时CPU编码**：在高质量设置（约31.5 dB PSNR，通道数n=20）下，仍可实现1080p 30fps的实时CPU编码。
- **极端压缩比**：支持超过5000:1的压缩比，而AVIF和JPEG分别只能达到352:1和139:1。
- **与mbt2018等更重编解码器比**：在0.1–1 bpp范围内，mbt2018虽然PSNR有+2.2~+4.2 dB优势，但其CPU编码吞吐量仅约0.16-0.17 MPx/s，比FRAPPE慢两个数量级，不适合边缘编码场景。
- **不对称部署有效**：将大多数计算量分配到解码端，使编码端保持极轻量，验证了不对称设计在受限传感器上的可行性。

## 7. 优点
- **零开销变速率与渐进式编码**：仅通过截断潜在通道即可切换码率，无需重编码或调节超参数，内存/计算开销极小。
- **极低编码复杂度**：编码器仅为几个步长卷积和逐元素非线性操作，计算量约68 ops/pixel，可并行化，适合CPU实时执行。
- **自然的通道重要性排序**：训练机制自动将信息量优先分配到前几个通道，符合率失真优化趋势。
- **不对称设计合理**：在资源受限的传感器侧承担极轻编码，在云端用强大解码器恢复高质量，契合边缘-云协同场景。
- **跨模态扩展潜力**：框架理论上适用于1D、2D、3D信号（音频、高光谱、视频等），具有通用性。
