## 摘要
尽管前馈式3D高斯喷溅方法能够从稀疏上下文视图中重建可渲染的高斯基元，而无需逐场景优化，但现有流程并未提供用于存储或传输的紧凑场景表示。一种自然的解决方案是将现有3DGS压缩方法应用于生成的高斯基元。然而，这种方法作用于最终的不规则3D表示，并且与内部特征到高斯的生成过程解耦，这限制了压缩效率。为解决这一问题，我们提出了CodecSplat，一种用于前馈式3D高斯喷溅的超紧凑潜在编码框架。CodecSplat首先将中间2D高斯生成特征编码为熵编码的场景比特流。在解码器端，潜在特征被重建并用于预测深度和高斯参数，然后将其映射到3D高斯基元。注意，通过将压缩集成到前馈高斯生成流程中，CodecSplat避免了对不规则3D高斯基元的低效压缩，并允许编解码器利用结构化中间特征表示。我们在一个具有深度引导多视图特征细化与分层学习特征编解码器的前馈高斯喷溅骨干网络上实例化了CodecSplat。在DL3DV和RealEstate10K数据集上，CodecSplat分别以每场景仅20.00–107.77 KiB和3.37–12.51 KiB的比特率实现了23.56–26.36 dB和24.76–27.05 dB的PSNR。这比压缩前馈生成的高斯基元大约小一个数量级，同时保持可控的率失真行为。

## Abstract
While feed-forward 3D Gaussian splatting reconstructs renderable Gaussian primitives from sparse context views without per-scene optimization, existing pipelines do not provide a compact scene representation for storage or transmission. A natural solution is to apply existing 3DGS compression methods to the generated Gaussian primitives. However, this approach operates on the final irregular 3D representation and is decoupled from the internal feature-to-Gaussian generation process, which limits compression efficiency. To address this, we introduce CodecSplat, an ultra-compact latent coding framework for feed-forward 3D Gaussian splatting. CodecSplat first encodes an intermediate 2D Gaussian-generation feature into an entropy-coded scene bitstream. At the decoder, the latent feature is reconstructed and used to predict depth and Gaussian parameters, which are then mapped to 3D Gaussian primitives. Note that, by integrating compression into the feed-forward Gaussian generation pipeline, CodecSplat avoids inefficient compression over irregular 3D Gaussian primitives and allows the codec to exploit the structured intermediate feature representation. We instantiate CodecSplat on a feed-forward Gaussian splatting backbone with depth-guided multi-view feature refinement and a hierarchical learned feature codec. On DL3DV and RealEstate10K datasets, CodecSplat achieves 23.56-26.36 dB and 24.76-27.05 dB PSNR with only 20.00-107.77 KiB and 3.37-12.51 KiB per scene, respectively. This is roughly one order of magnitude smaller than compressing feed-forward generated Gaussian primitives, while preserving controllable rate-distortion behavior.

- **研究动机**：观察到在FF3DGS中，在生成最终高斯基元之前，网络已经将场景信息组织为密集、视图对齐、具有几何感知的2D网格特征。如果在这个中间阶段进行压缩，可以避免处理不规则3D基元，从而大幅提高压缩效率。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将编码瓶颈置于2D高斯生成特征（即从多视图特征中提取的中间表示）上，而不是在最终3D高斯基元上。该特征仍是规则、视图对齐的张量，经过熵编码后形成场景比特流；解码端重建特征，然后预测深度和高斯参数，再2D到3D映射得到高斯基元。
- **关键技术细节**：
  1. **深度引导的多视图特征细化**：在编码前和解码后，通过基于深度的跨视图特征 warp 和平均聚合，增强几何感知和跨视图冗余消除，分别得到预编码特征Fc和重建特征Fdec。
  2. **分层学习特征编解码器**：使用层次化变分自编码器（VAE）结构，对Fc进行分析变换、量化、熵编码（基于上下文模型），并通过率参数λ控制不同率-失真折中点。解码端通过合成变换重建Fc。
  3. **深度与高斯参数解码**：从重建特征Fdec预测归一化逆深度，利用比特流中的近远边界恢复度量深度；然后高斯预测头输出每个像素对齐的属性（位置、协方差、不透明度、球谐系数等），结合相机参数映射到3D空间。
  4. **两阶段优化**：
     - 第一阶段：无编解码器训练，仅优化含深度引导细化的高斯生成骨干（冻结官方DepthSplat初始权重），训练100K步。
     - 第二阶段：启用编解码器，固定骨干参数，仅训练编解码器相关模块（包括后细化器、深度头、高斯头等），训练200K步，损失函数包含渲染损失、深度监督损失和率损失。
- **公式/算法流程（文字说明）**：
  1. 输入N个上下文视图，经MVEnc提取多视图特征Fmv、深度预测Denc等。
  2. 通过FeatHead生成高斯生成特征Fg。
  3. 深度引导细化（预编码）：Fc = Fg + Rpre(Fg, A(Fg, Denc), Denc^{-1})，其中A为跨视图对齐聚合。
  4. 编解码：Z = Analysis(Fc; eλ), ˆZ=Q(Z), ˆFc=Synthesis(ˆZ; eλ)。
  5. 后细化：Fdec = ˆFc + Rpost(ˆFc, A(ˆFc, ˆD), ˆD^{-1})，其中ˆD来自解码深度头。
  6. 从Fdec预测深度和所有高斯属性，经过Map2D→3D转换为3D高斯基元，最后渲染目标视图。

## 6. 论文的主要结论与发现

- CodecSplat在DL3DV上以20.00-107.77 KiB/场景达到23.56-26.36 dB PSNR；在RE10K上以3.37-12.51 KiB/场景达到24.76-27.05 dB PSNR。

- 深度引导的多视图细化将无编解码器PSNR从25.68 dB提升至26.22 dB，验证了跨视图交互的有效性。

## 7. 优点

- **创新性**：首次将压缩瓶颈放在FF3DGS的中间2D特征上，避免了不规则3D基元的低效压缩，提出集成式压缩范式。
- **紧凑性**：实现了KB级别的场景表示，显著小于现有方法（MiB级别），便于存储和传输。
- **可控率失真**：通过单模型调整λ即可覆盖不同比特率，无需重新训练。
- **效率**：压缩/解压缩时间远小于大多数基线方法（如SOGS需数十秒），且GS生成时间优于DepthSplat和ReSplat。
- **消融完备**：清晰验证了每个设计选择（深度细化、学习编解码器）的必要性和贡献。

## 8. 不足与局限

- **重建质量受限**：PSNR上界受限于底层的DepthSplat风格骨干，低于ReSplat（28.96 dB）。因此CodecSplat的最高PSNR（26.36 dB）仍低于ReSplat+压缩基线的28.72 dB（但代价是数MiB的比特流）。未来需探索更强骨干。
- **需要联合训练**：编解码器不能直接插入任意预训练FF3DGS模型，必须与生成流水线联合训练第二阶段，限制了通用性。
- **实验覆盖有限**：仅测试了两个标准数据集（DL3DV和RE10K），未在更复杂场景（如大户外场景、动态场景）下验证；也未与更新的压缩方法（如HAC、ContextGS、CompGS等）在统一设置下对比（但那些方法针对逐场景优化，而非前馈设置）。
- **应用限制**：当前框架只支持前馈式GS，无法用于传统逐场景优化的3DGS；且编解码器仅针对多视图输入（N=2或8），未探讨单视图或任意视图数情况。
- **未报告泛化性**：训练和测试场景可能来自相同数据分布，跨数据集泛化能力未知。
