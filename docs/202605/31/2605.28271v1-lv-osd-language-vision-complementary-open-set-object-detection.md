## 摘要
目标检测是计算机视觉中的一项重要任务，旨在通过给定的类别列表或查询图像检测感兴趣的目标。本文提出一个新的问题——语言-视觉-互补开放集目标检测（LV-OSD），即使用灵活的基于文本和/或图像的提示来指定所需的目标类别。这一设置在实际应用中更为常见和实用。为此，我们设计了一个双分支检测框架LVDor，能够同时接受文本和图像提示。具体地，我们首先构建包含每个类别的多种文本描述和图像样本的多模态提示（MPr）。随后，为弥合输入图像、文本提示和图像提示之间的语义鸿沟，我们设计了目标引导的提示动态加权（TPDW）模块。该模块在目标图像先验信息的引导下，动态生成与目标语义最匹配的文本和图像提示，实现精确对齐并有效减少两种模态之间的差异，从而适应LV-OSD设置。我们还提出一种简单的训练时提示随机掩码（PRM）机制，以模拟测试中文本和/或图像提示的任意组合。大量实验结果验证了问题定义的合理性以及方法的有效性。提示和代码将公开发布。

## Abstract
Object detection is an important task in computer vision, which aims to detect the objects of interest. through the given category list or query images. In this work, we propose a new problem of language-visual-complementary open-set object detection (LV-OSD), i.e., using the flexible text-based and/or image-based prompts to specify the desired object categories. This setting is more common and practical in real-world applications. For this purpose, we design a dual-branch detection framework, LVDor, which can simultaneously accept both text and image prompts. Specifically, we first build the Multi-modal Prompts (MPr) containing various text descriptions and image samples for each category. Subsequently, to bridge the semantic gap among the input image, text prompts, and image prompts, we design a Target-guided Prompt Dynamic Weighting (TPDW) module. Guided by the prior information of the target image, this module dynamically produces the text and image prompts that best align with the target semantics, achieving precise alignment and effectively reducing the discrepancy between the two modalities, thereby accommodating the LV-OSD setting. We also propose a simple Prompt Random Masking (PRM) mechanism during training to simulate the arbitrary combination of text and/or image prompts in testing. Extensive experimental results verify our problem formulation's reasonability and our method's effectiveness. Prompts and code will be released publicly.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：现有的开放集目标检测（OSD）主要分为两类：基于文本提示的开放词汇检测（OVD）和基于图像示例的少样本/单样本检测。前者依赖手工制作的文本模板（如“a photo of a dog”），存在语义歧义（如同音词“bat”）、无法捕捉细粒度视觉细节、对某些稀有类别难以用文本准确描述等局限；后者利用图像示例提供更丰富的视觉信息，但单个图像难以覆盖类内多样性，且某些类别难以获取代表性样本。
- **核心动机**：现实场景中，用户可能同时拥有文本类别名称、图像示例或两者混合，而现有方法要求所有类别采用统一的提示模态（全文本或全图像），缺乏灵活性。因此，作者提出一个新的问题设定——语言-视觉-互补开放集目标检测（LV-OSD），允许在推理时任意组合文本、图像或两者混合的提示，以适应实际应用中的多样化需求。
- **意义**：LV-OSD 首次定义了支持三种以上模态组合（纯文本、纯图像、融合提示、灵活混合提示）的开放集检测任务，更贴近真实使用场景，提升了部署的实用性。

## 2. 方法论

### 核心思想
设计一个双分支检测框架 **LVDor**，能够同时接收文本和图像提示，并通过动态对齐机制弥合两者与输入图像之间的语义鸿沟，利用提示随机掩码策略使模型适应任意模态组合。

### 关键技术细节

- **多模态提示构建（MPr）**：
  - **文本提示**：使用大语言模型 DeepSeek 为每个类别生成 5 种不同风格的互联网图像标题式描述，强调细粒度视觉属性（如形状、纹理、场景），以覆盖类内多样性。
  - **图像提示**：从互联网为每个类别收集多张示例图像，涵盖不同光照、角度、背景、姿态等变化。
  - 所有提示通过冻结的 CLIP 文本/图像编码器提取嵌入向量，构成统一的 MPr 集合。

- **目标引导的提示动态加权（TPDW）**：
  - **动机**：同一类别的多个文本/图像提示与输入图像中目标对象的语义一致性不同，直接平均会导致语义噪声。
  - **步骤**：
    1. 将输入图像的特征图划分为 4 个 patch，对每个 patch 计算其与所有类别平均提示的余弦相似度，选取前 K 个候选类别（K=5）。
    2. 对每个候选类别，计算该 patch 特征与类别内每个提示的相似度，经 softmax 得到权重，加权后得到该 patch 下的类别提示。
    3. 汇总所有 patch 的结果：若某类别被多个 patch 加权，则取平均；否则使用原始平均提示。
    4. 同理处理图像提示。
  - **作用**：动态选择与输入图像语义最相关的提示，抑制类内不一致的提示，实现精确跨模态对齐。

- **提示随机掩码（PRM）**：
  - **训练时**：随机将文本提示或图像提示掩码为全零向量（即丢弃一种模态），迫使模型学习只依赖单一模态或任意组合。具体地：$ \hat{\mathbf{P}}_{\text{text}} = \mathbf{P}_{\text{text}} \odot \mathbf{M}_{\text{text}}, \quad \hat{\mathbf{P}}_{\text{img}} = \mathbf{P}_{\text{img}} \odot \mathbf{M}_{\text{img}}$，其中 $\mathbf{M}$ 为随机二进制掩码。
  - **测试时**：根据用户实际提供的提示（可能是纯文本、纯图像或混合），直接使用相应的可用提示，模型通过训练学会处理任意缺失。
  
- **检测框架**：
  - 基于 Faster R-CNN 架构，使用冻结的 CLIP ViT 作为特征提取器（文本编码器+图像编码器），训练 RPN 和 RoI 头部。
  - 分类头部（MM-Cls）采用对比损失（L_cons），将 RoI 特征与经 TPDW 调整后的多模态提示嵌入进行余弦相似度匹配。
  - 总损失：$L_{\text{total}} = L_{\text{reg-RPN}} + L_{\text{cls-RPN}} + L_{\text{reg-RoI}} + L_{\text{cons}}$。

## 6. 主要结论与发现

- **问题定义的合理性**：LV-OSD 能够支持纯文本、纯图像、融合及任意混合提示，实验显示统一训练后的模型在所有模态组合下均取得不错结果，且明显优于直接套用单一模态训练的方法。
- **方法有效性**：
  - LVDor 在 OV-LVIS 上达到 SOTA 性能（如 DeCLIP ViT-L/14 下 AP=38.4%，融合提示）。
  - 在柔性提示设置下，LVDor 的精度下降远小于对比方法（如 PRM 关键作用）。
  - 跨数据集迁移至 Objects365 同样取得最优或相当结果。
- **关键技术贡献**：
  - TPDW 模块通过动态选择提示有效弥合模态语义鸿沟。
  - PRM 机制使模型学会处理任意模态缺失，是柔性提示成功的关键。

## 7. 优点

- **问题创新性**：首次提出 LV-OSD 任务，填补了开放集检测中灵活多模态提示的空白。
- **方法实用性**：LLM 生成丰富文本描述、互联网挖掘多样化图像提示，无需人工精细标注。
- **模块设计巧妙**：
  - TPDW 利用输入图像先验动态加权，避免简单平均导致的语义噪声。
  - PRM 简单有效，将不确定性纳入训练，增强了模型鲁棒性。
- **实验扎实**：多数据集、多模态设置、多种 backbone、充分消融和敏感度分析，且与主流方法公平对比。
- **代码和提示将开源**，便于复现和落地。

## 8. 不足与局限

- **依赖 CLIP backbone**：框架基于冻结的 CLIP，其表达能力受限于预训练模型；在严重领域偏移下可能无法取得进一步提升。
- **计算开销**：TPDW 增加了推理时间（13.61 FPS vs 21.18 FPS），离实时应用仍有差距。
- **域迁移限制**：虽在 Objects365 表现良好，但该方法并未在更多样化（如卫星图、医学图像）场景验证。
- **训练数据依赖**：需要为每个类别收集图像示例并生成文本描述，对于极大量类别（如 20K）可能成本增加。
- **未探索更轻量化架构**：论文仅基于 ViT-B/16 和 ViT-L/14，未尝试更轻量的 backbone 或蒸馏方法。
- **未报告所有对比方法的算力与效率**：仅对部分方法测速，公平性有待进一步完善（如 OV-DQUO 和 MM-OVOD 的速度对比）。
